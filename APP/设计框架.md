# APP 设计框架

PC 端工具（`APP/serial_ota_tool`）的分层结构与扩展方式。

## 目录分层

```
APP/
├── 设计框架.md              ← 本文
├── README.md
├── test/                    # 只读参考（勿改）
└── serial_ota_tool/
    ├── main.py              # 入口：创建 QApplication，加载主窗口
    ├── app/                 # 界面与交互（PyQt5 Widgets）
    │   └── mainwindow.py    # 主窗口、标签页、信号槽
    ├── core/                # 与 UI 无关的业务逻辑
    │   ├── serial_io.py     # 串口读写封装
    │   ├── serial_thread.py # 串口读线程（QThread）
    │   ├── ota_client.py    # HTTP /api/info、/api/ota
    │   └── hex_util.py      # 十六进制显示等
    ├── ui/                  # 纯视觉
    │   └── theme.py         # 主题（白蓝 / 黑金 / 科技蓝）
    ├── build_exe.ps1        # PyInstaller 单次打包（手动执行）
    └── requirements.txt
```

| 层 | 职责 | 可依赖 |
|----|------|--------|
| `ui/` | 颜色、字体、QSS，无业务 | 仅 PyQt5 |
| `core/` | 串口、OTA、协议、线程 | 标准库 + requests/serial |
| `app/` | 布局、用户操作、连接 core 信号 | `core/`、`ui/` |
| `main.py` | 启动顺序、全局异常 | `app/` |

扩展新功能时：协议与 IO 放 `core/`，新页面或控件放 `app/`（如 **设置** 标签页），换肤只改 `ui/theme.py`。

## 主界面标签页

| 标签 | 内容 |
|------|------|
| 串口调试 | 端口、收发、日志 |
| OTA 升级 | 设备查询、网段扫描、固件上传 |
| 设置 | 界面主题等 |

## 本地运行与打包

```powershell
cd APP/serial_ota_tool
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

需要 EXE 时手动执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\build_exe.ps1
```

输出：`dist\ESP32-C3-Serial-OTA.exe`
